Prompt-based Learning in NLP

Posted on 2022-06-22 In paper reading

This blog is based on: Liu et al. (2021) Pre-train, Prompt, and Predict: A Systematic Survey of Prompting Methods in Natural Language Processing.

Introduction

In a traditional supervised learning system (e.g. text classification or machine translation), we take an input x, and predict an output y based on a model \(P(\textbf{y}|\textbf{x};\theta)\). The main issue with supervised learning tasks is the requirement of a large amount of annotated labels, which are usually not available for many tasks. This can be addressed by prompt-based learning, which attempts to learn the probablity \(P(\textbf{x};\theta)\) and use this probability to predict y instead of learning \(P(\textbf{y}|\textbf{x};\theta)\) directly.

Score matching methods

Posted on 2021-11-06 Edited on 2022-06-22

Introduction

The calculation of score function \(\nabla{\textbf{x}}\ \mathrm{ log} p(\textbf{x})\) is the key in score-base generative models. To estiamte the score function, we train a score-based model \(\textbf{s}_{\theta}(\textbf{x})\) by minimizing the Fisher divergence: \[\mathbb{E}_{p(\mathbf{x})}\left[\left\|\nabla_{\mathbf{x}} \log p(\mathbf{x})-\mathbf{s}_{\theta}(\mathbf{x})\right\|_{2}^{2}\right].\] Direct computation of this loss function, however, is not feasible due to the lack of the knowledge of the ground-truth data score function \(\nabla{\textbf{x}}\ \mathrm{ log} p(\textbf{x})\). Instead the loss function is minimized using score matching methods where the knowledge of ground-truth data score function is not required. In this post, I will introduce denoising score matching and sliced-socre matching method.

Score-based generative modeling

Posted on 2021-11-02 Edited on 2022-06-22 In paper reading

This blog is based on: Song (2021): Generative Modeling by Estimating Gradients of the Data Distribution.

Introduction

Popular generative models can be grouped into two categories:

  1. likelihood-based models include autoregressive models, energy-based models (EBMs) normalizing flows and variational autoencoders (VAEs). These models learn the probability function directly.
  2. implicit generative models such as GANs. These models represent the probability distribution implicitly by a model of a sampling process.

Desipite of the greate success of these models, each has the limitations of its own. For example, flow-based models require a care design of architectures to construct reversible transform. EBMs need to have a tractable normalization constant. VAE relies on a surrogate loss (ELBO). GAN models are known for potentially unstable training and mode collapse problems.

These problems can be addressed by score-based generative models, which represent probability distribution via the score function.
